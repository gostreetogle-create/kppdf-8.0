ШАГ 1: ImportJobs (импорт данных)

- `POST /import-jobs` (multipart) — загрузить CSV/Excel файл, создать документ со статусом 'pending'.
- `POST /import-jobs/:id/start` — запустить обработку (в фоне через `@nestjs/bull` или простой `setImmediate`):
  1. Парсинг CSV (`csv-parse`) или Excel (`xlsx`).
  2. `progressPercent = 0`, `processedRecords = 0`.
  3. Для каждой строки — попытка создать документ `entityType` (Material, Product, Counterparty — в зависимости от задачи).
  4. `progressPercent = (processedRecords / totalRecords) * 100`, обновлять каждые N строк.
  5. При ошибке — записать в `errorLog`, `failedRecords++`.
  6. По завершении — `status: 'completed' | 'failed'`, `completedAt = new Date()`.
- `GET /import-jobs?status=...&entityType=...`.
- `GET /import-jobs/:id` с populate createdByUserId.
- `POST /import-jobs/:id/cancel` (если в фоне).
- `DELETE /import-jobs/:id` — soft-delete.

**Предустановленные шаблоны импорта:**
- `materials-template.csv` — колонки: name, article, sku, unit, pricePerUnit, categoryName, supplierInn.
- `products-template.csv` — колонки: name, sku, kind, unit, listPrice, categoryName.
- `counterparties-template.csv` — колонки: name, shortName, inn, kpp, ogrn, phone, email.

ШАГ 2: Comment (универсальные комментарии)

```typescript
{ packageTag: string; authorId: ObjectId → User; author?: string (кэш); text: string; isArchived: boolean }
```

- `POST /comments {packageTag, text}` — создать (authorId из JwtAuthGuard).
- `GET /comments?packageTag=...&isArchived=false` — все комментарии для пакета сделок (например, `KP-2026-001`).
- `PATCH /comments/:id` — обновить text (только автор или admin).
- `POST /comments/:id/archive` — `isArchived = true`.
- `DELETE /comments/:id` — soft-delete.
