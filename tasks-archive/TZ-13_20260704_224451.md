ШАГ 1: Quotation (CRUD, консолидация Proposal+CommercialProposal)

**Канонические поля (после консолидации):**
- `number`, `organizationId` (наша компания), `counterpartyId`, `tenderId?`, `date`, `validUntil`, `statusId`, `total`, `discountType`, `discountPercent`, `discountAmount`, `notes`, `isActive`, `templateId` (→ DocumentTemplate из TZ-14), `designSnapshot`, `templateSnapshot`, `items[]`.

**Endpoints:**
- `GET /quotations` — список с пагинацией, фильтр по counterparty, status, date range.
- `GET /quotations/:id` — populate items.product, counterparty, organization.
- `POST /quotations` — создать с items.
- `PATCH /quotations/:id`.
- `POST /quotations/:id/duplicate` — копировать (для новой версии).
- `POST /quotations/:id/convert-to-contract` → создать Contract.
- `POST /quotations/:id/convert-to-order` → создать Order.

**Snapshot-поля в items:**
- `unitPrice`, `markupPercent`, `total` сохраняются на момент создания (не вычисляются динамически из Product). Это критично для ERP — цена продукта может измениться, но в КП остаётся та, по которой договорились.

ШАГ 2: Contract (CRUD)

- `POST /contracts` — на основе Quotation (или standalone).
- `GET /contracts?counterpartyId=...&status=...&from=...&to=...`.
- `POST /contracts/:id/sign {signedAt}` → status = 'signed', `signedAt` сохраняется.
- `POST /contracts/:id/activate` → создать Order на основе Contract.

ШАГ 3: Order (CRUD)

- `POST /orders` — `{counterpartyId, quotationId?, items: [{productId, quantity}], deliveryAddress, managerId, priority}`.
- Snapshot-поля (item.unitPrice = текущая цена Product).
- `GET /orders?counterpartyId=...&status=...&managerId=...`.
- `POST /orders/:id/reserve-stock` — создать Reservations (TZ-11) для всех items.
- `POST /orders/:id/ship` — создать Shipment.
- `POST /orders/:id/cancel` — освободить Reservations.

ШАГ 4: Shipment (CRUD)

- `POST /shipments` — на основе Order.
- `GET /shipments?orderId=...&status=...&date=...`.
- `POST /shipments/:id/dispatch` — создать `out` StockMovements (TZ-11), освободить Reservations.
- `POST /shipments/:id/add-doc {type, totalAmount, pdfUrl}` — создать ShippingDoc.

ШАГ 5: ShippingDoc (CRUD)

- Стандартный CRUD вложенно в Shipment.
- `GET /shipping-docs/:id/download?format=pdf` — отдать PDF (если реализовано; иначе — просто отдать URL).

ШАГ 6: CartItem + CartSession

- `POST /cart-sessions` — создать сессию (для анонимной корзины, без auth). Возвращает `{sessionId, createdAt}`.
- `POST /cart-items` — `{sessionId, productId, quantity, markupPercent?}` — `priceSnapshot` = текущая цена Product. Анонимный endpoint.
- `GET /cart-sessions/:id/items` — список с populate product. Анонимный.
- `PATCH /cart-items/:id` — `{quantity}`.
- **`POST /cart-sessions/:id/checkout`** — конвертировать в Quotation. **Требует авторизации** (JwtAuthGuard). Варианты:
  - **Вариант A (existing customer):** пользователь уже залогинен → `{organizationId, counterpartyId}` (обязательно) → создаётся Quotation со snapshot-полями из CartItems.
  - **Вариант B (new customer):** пользователь ещё не залогинен → 401 + сообщение "Требуется login или регистрация". Сначала `POST /auth/login` или `POST /auth/register`, затем `POST /cart-sessions/:id/checkout`.
  - **Вариант C (guest checkout):** НЕ поддерживается в TZ-13 (требует guest-логики и анонимной Counterparty, отложить).
- При checkout: создаётся Quotation со статусом `draft`, CartItems помечаются как `converted` (добавить `isConverted: boolean` в CartItem).
