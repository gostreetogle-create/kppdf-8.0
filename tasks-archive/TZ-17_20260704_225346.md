ШАГ 3: E2E тесты (5 критических flow)

**`auth.e2e-spec.ts`:**
- `POST /auth/register` — создать User, проверить 201 + `{access, refresh, user}`.
- `POST /auth/login` — с правильным паролем → 200, с неправильным → 401.
- `GET /auth/me` с access token → 200, без токена → 401.
- `POST /auth/refresh` → 200, с истёкшим → 401.
- `POST /auth/logout` → старый refresh → 401.

**`products.e2e-spec.ts`:**
- `POST /products` (admin) → 201, проверить автогенерацию `sku`.
- `POST /products/:id/photos` (multipart) → 201, файл сохранён.
- `GET /products?categoryId=...&search=...` — фильтр работает.
- `POST /products/:id/boms` → 201, `GET /boms/:id/expanded` — populated.
- `POST /attribute-definitions` + `POST /products/:id/attributes` — EAV работает.

**`quotations.e2e-spec.ts`:**
- `POST /quotations` — 201, snapshot-поля сохранены.
- `POST /quotations/:id/duplicate` — новая Quotation.
- `POST /quotations/:id/convert-to-contract` — Contract создан.
- Snapshot: после `PATCH /products/:id` (изменение цены), старая Quotation.items[0].unitPrice не изменилась.

**`orders.e2e-spec.ts`:**
- `POST /orders` → 201, snapshot-поля.
- `POST /orders/:id/reserve-stock` → Reservations созданы, проверка `available >= requested`.
- `POST /orders/:id/ship` → Shipment создан.
- `POST /orders/:id/cancel` → Reservations освобождены.

**`warehouse.e2e-spec.ts`:**
- `POST /stock-movements {type: "in", qty: 100}` → StorageItem.quantity = 100.
- `POST /stock-movements {type: "out", qty: 200}` → 400 (недостаточно).
- `POST /stock-movements {type: "transfer", qty: 30}` → транзакция, оба StorageItem обновлены.
- `GET /inventory/low-stock` — фильтр работает.

**`production.e2e-spec.ts` (сквозной production-flow):**
1. Login как admin.
2. `POST /products` (создать тестовый продукт + TechProcess с 3 операциями).
3. `POST /production-orders {productId, quantity: 5, ...}` → 201, **транзакционно** созданы OrderTask[].
4. `GET /production-orders/:id/tasks` — все tasks созданы, `dependsOnTaskIds` корректны.
5. `POST /order-tasks/:task1Id/complete` → 200; `POST /order-tasks/:task2Id/complete` (зависит от task1) → 200.
6. `POST /order-tasks/:task2Id/complete` повторно → 400 (уже done).
7. `POST /products/:id/cost-calculations` → возвращает `{totalMaterialCost, totalLaborCost, totalCost}`.
8. `POST /production-orders/:id/actual-costs {type: "material", amount: 5000}` → 201.
9. `GET /production-orders/:id/cost-comparison` → `{planned, actual, variance}`.
10. `POST /production-orders/:id/close {closingType: "completed", number, date, amount}` → проверяет что все tasks completed, создаёт OrderClosing.
11. `POST /work-orders {orderTaskId, qty, assignedTo}` → 201, WorkOrderOperation[] созданы.
12. `POST /work-order-operations/:id/start` → `startedAt` заполнен. `POST /work-order-operations/:id/complete {actualDuration: 2}` → `completedAt` заполнен.
13. AuditLog содержит записи для всех шагов (verify через `GET /audit-logs?entityType=ProductionOrder&entityId=...`).

**`integration.e2e-spec.ts` (сквозной):**
1. Login как admin.
2. `POST /counterparties` — создать контрагента.
3. `POST /products` — создать продукт.
4. `POST /quotations {counterpartyId, items: [{productId}]}` — КП.
5. `POST /quotations/:id/convert-to-contract` → Contract.
6. `POST /contracts/:id/sign {signedAt}`.
7. `POST /contracts/:id/activate` → Order.
8. `POST /orders/:id/reserve-stock` → Reservations.
9. `POST /orders/:id/ship` → Shipment + StockMovements.
10. `POST /invoices {supplierId: <counterpartyId>, totalAmount: ...}` → Invoice.
11. `GET /reconciliation-acts/generate` → ReconciliationAct со всеми расчётами.
12. Проверить AuditLog — должны быть записи для всех действий.
