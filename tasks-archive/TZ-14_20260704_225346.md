ЧТО ДЕЛАТЬ
═══════════════════════════════════════════════════════════════

ШАГ 1: DocType (CRUD, консолидация DocTypeDef)

- Стандартный CRUD.
- Seed: дефолтные типы — `proposal`, `contract`, `order`, `invoice`, `act`, `report`.

ШАГ 2: DocumentTemplate (CRUD)

- `POST /document-templates` — `{name, description, tags, organizationId (→ Organization, TZ-06), docTypeId (→ DocType), isDefault, pageSize, backgroundImage[]?, blocks: [...], version}`.
- `GET /document-templates?organizationId=...&docTypeId=...&isDefault=true` — фильтр (organizationId обязателен: шаблон принадлежит конкретной организации).
- `GET /document-templates/:id/expanded` — populate blocks, docType, organization.
- `POST /document-templates/:id/duplicate` — копировать (новая версия).
- `POST /document-templates/:id/set-default` — сделать дефолтным для docType в данной организации (деактивировать другие `isDefault: true` для того же `(organizationId, docTypeId)`).
- При `POST /document-templates` с `isDefault: true` — деактивировать другие `isDefault: true` для того же `(organizationId, docTypeId)`.

ШАГ 3: TemplateBlock (CRUD, вложенно в DocumentTemplate)

- `POST /document-templates/:id/blocks` — `{type, order, title?, content?, height?, showLine?, settings?}`.
- `PATCH /template-blocks/:id` — обновить.
- `DELETE /template-blocks/:id` — soft-delete.
- `POST /document-templates/:id/blocks/reorder` — `{blockIds: [...]}` — изменить порядок.

ШАГ 4: TableTemplate (CRUD)

- Стандартный CRUD.
- `columns[]` — `[{key, label, width, align, format?}]`.
- Используется в `DocumentTemplate.blocks` через `type: 'table'` (settings.tableTemplateId ссылается на TableTemplate).

ШАГ 5: DocumentTableType (CRUD)

- Стандартный CRUD.
- `dataSource` — `'products' | 'orders' | 'proposals' | ...` (для автогенерации таблицы из БД).
- Используется в `DocumentTemplate.blocks` через `type: 'table'` (settings.documentTableTypeId ссылается на DocumentTableType).

ШАГ 6: PDF generation (базовая)

- `POST /document-templates/:id/preview?dataId=...` — отрендерить шаблон с данными, вернуть HTML или PDF (через `puppeteer` или `pdfkit` — `pnpm add puppeteer`).
- Поддерживаемые `dataId`:
  - `<quotation-id>` (Quotation из TZ-13) — для шаблонов `docType: 'proposal'`
  - `<contract-id>` (Contract из TZ-13) — для шаблонов `docType: 'contract'`
  - `<order-id>` (Order из TZ-13) — для шаблонов `docType: 'order'`
- Для TZ-14 достаточно HTML preview (PDF — опционально). Если PDF не реализован, endpoint возвращает HTML.
