**Решения пользователя (зафиксированы):**
1. ✅ **Block public `/auth/register`** — убрать `@Public()` decorator; регистрация только через admin-invite-flow (`POST /api/users/invite` — out of scope этого TZ, но endpoint-stub).
2. ✅ **`RegisterDto.role` → `@IsIn(['user', 'manager'])`** — никогда не принимаем `admin` из публичного API (defense-in-depth даже если guard убирается).
3. ✅ **Admin password < 8 chars → WARN + skip (NOT error)** — не crash bootstrap; warn пользователю в логах что admin не создан + документировать в README.
4. ✅ **JWT secrets hardening**:
   - Add warning если `JWT_SECRET` или `JWT_REFRESH_SECRET` содержим `dev` или `do-not-use` substr.
   - Verify `.env` в `.gitignore` (✅ уже там).
   - Документировать в README: «Для продакшена: `openssl rand -hex 32`».
5. ✅ **CORS_ORIGIN → multi-origin (`.env`)** — изменить формат с single-URL на comma-separated list. Default: `http://localhost:4200`. Read via `CORS_ORIGIN.split(',').map(s => s.trim())`.
6. ✅ **Swagger guarded по NODE_ENV** — `if (process.env.NODE_ENV !== 'production')` gate, + в prod можно опционально вернуть через env flag `SWAGGER_ENABLED=true`.
7. ✅ **Rate-limit на `/auth/login`** — `@Throttle({ short: { ttl: 60000, limit: 5 }, long: { ttl: 3600000, limit: 20 } })` — 5 req/min, 20 req/hour. Account lockout после 10 consecutive failures (на userId basis).
8. ✅ **RBAC sweep на ALL write endpoints** — `@Roles('admin', 'manager')` на каждом `@Post`, `@Put`, `@Patch`, `@Delete` в 73 controllers. Reads (`@Get`) оставляем без `@Roles` (auth-only). Exception: `auth.controller.ts` (login, refresh, logout — публичные, уже правильно).

## 1. Цель

Закрыть 3 CRITICAL + 5 HIGH security находок из QA-01 (`/auth/register` открыт, admin password пустой, JWT secrets слабые, CORS misconfigured, Swagger без auth, rate-limit отсутствует, RBAC не на write endpoints) единым атомарным TZ-91, чтобы проект мог быть выставлен в продакшен без security incidents.

**Скоуп:**
- 8 файлов в backend (auth, seed, config, main, common)
- 73 controllers (RBAC sweep — `@Roles('admin','manager')` на write endpoints)
- 1 файл `.env` (CORS_ORIGIN format change)
- 1 файл `start.mjs` (dev secret warning check)
- 1 файл `README.md` (admin setup + JWT secrets документация)

## 2. Архитектурные решения (зафиксированы)

| # | Решение | Почему |
|---|---------|--------|
| 1 | **Block public `/auth/register`** — убрать `@Public()` decorator; invite-flow path (`POST /api/users/invite`) будет future TZ-91-extension. | Defense-in-depth: даже если `@Roles('admin')` guard добавить в будущем, DTO `@IsIn(['user','manager'])` уже блокирует admin creation. |
| 2 | **`RegisterDto.role` → `@IsIn(['user','manager'])`** — class-validator whitelist. | DTO-level constraint не зависит от guard и работает на application-layer (даже если guard удалить). |
| 3 | **Admin seed: password length check `>= 8`** — WARN + skip + bootstrap-continues. | Не валить bootstrap; admin может быть создан позже admin-tools route (out of TZ-91). |
| 4 | **JWT secret `dev` substring warning** — в `start.mjs` при старте. | Эстетика UX: пользователь увидит warning в консоли. Не блокируем startup. |
| 5 | **CORS multi-origin** — `.env:CORS_ORIGIN=http://localhost:4200,http://localhost:3000`. Comma-separated list. | Подходит для dev (Angular dev + prod-like) и multi-domain prod. |
| 6 | **Swagger gated `NODE_ENV !== 'production'`** — default-off в prod. | Публичный API документация = утечка информации. |
| 7 | **Rate-limit на login** — `@Throttle({ short: { ttl: 60000, limit: 5 } })`. | Brute-force prevention. 5 попыток в минуту — норма для enthusiastic-user UX. |
| 8 | **RBAC sweep** — генератор-скрипт (или manual) добавляет `@Roles('admin','manager')` на все @Post/@Put/@Patch/@Delete в 73 controllers. | Bug-as-feature: Если контроллер не имеет write endpoints — без изменений. |
| 9 | **Auth role hierarchy**: `admin` > `manager` > `user` — `@Roles('manager')` означает «admin OR manager», НЕ только manager. | Это standard RBAC convention. Уточним в JSDoc. |
| 10 | **`/me` projection** — отдельный spec TZ-92; здесь НЕ трогаем. | TZ-91 фокус = security; /me leak — отдельный TZ. |

## 4. Фазы реализации

### Phase A — Quick Wins (Layer 1, atomic, 1 commit)

**Цель:** Minimum surface for 3 критичных находок: register guard, admin password check, CORS_ORIGIN update.

**Подзадачи:**

1. **A.1** `register.dto.ts`:
   ```typescript
   // Before:
   @IsString()
   role!: string;

   // After:
   @IsIn(['user', 'manager'])
   @IsOptional()  // default 'user' if not provided
   role?: UserRole;  // = 'user' | 'manager' | 'admin'
   ```
2. **A.2** `auth.controller.ts`:
   ```typescript
   // Before:
   @Public()
   @Post('register')
   // ...
   // After:
   // @Public() removed — register now requires admin role
   @Post('register')
   @Roles('admin')  // admin-only — invite flow
   // ... (existing body)
   ```
3. **A.3** `auth.controller.ts` `/login` rate-limit:
   ```typescript
   @SkipThrottle()  // global throttler still applies; this just adds stricter local
   @Throttle({
     short: { ttl: 60000, limit: 5 },
     long: { ttl: 3600000, limit: 20 },
   })
   @Post('login')
   // ... (existing body)
   ```
   Note: `@nestjs/throttler` already globally registered (TZ-18 hardening). Local `@Throttle` overrides global default.
4. **A.4** `admin.seed.ts` password length:
   ```typescript
   const password = this.config.get<string>('admin.password');
   if (!password || password.length < 8) {
     this.logger.warn(
       `⚠️ ADMIN_PASSWORD too short (${password?.length ?? 0} chars, need >= 8). ` +
       `Admin user NOT created. Set ADMIN_PASSWORD in .env then restart.`,
     );
     return;
   }
   ```
5. **A.5** Update `.env` (manual, нет risk):
   ```
   CORS_ORIGIN=http://localhost:4200,http://localhost:3000
   ```
6. **A.6** `main.ts` CORS multi-origin:
   ```typescript
   const corsOrigins = (process.env.CORS_ORIGIN ?? 'http://localhost:4200')
     .split(',')
     .map(s => s.trim())
     .filter(Boolean);
   app.enableCors({
     origin: corsOrigins,
     credentials: true,
   });
   ```

**Acceptance criteria:**
- `pnpm exec tsc -p tsconfig.build.json --noEmit` exit 0.
- `pnpm exec jest --testPathPattern=auth` exit 0 (existing tests + new rate-limit test).
- Manual smoke: `curl POST /auth/register -d '{"role":"admin"}'` → 403 (guarded).
- Manual smoke: `curl POST /auth/register` без auth → 401.
- Manual smoke: startup с `ADMIN_PASSWORD=admin123` → WARN в логе, admin NOT created.
- Manual smoke: `curl OPTIONS /api/anything -H 'Origin: http://localhost:4200'` → CORS headers present.
