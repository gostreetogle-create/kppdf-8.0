## 2. Root cause analysis

1. **Inventory fragmentation**: 7 pages × 1 sub-task each = 7 small commits. Mechanical, low-risk per page, but loses cross-page pattern reuse that batch-1 gained. **Pattern-axis grouping** (4 sub-patterns from §1) lets each sub-TZ-N document ONE pattern reapplied verbatim across N pages of that pattern.
2. **Per-page quirks** (lookup tables, status cycles, photo galleries): Different from materials/products/orders. Documented per-page inside each sub-TZ-N (column + cellTemplate decisions).
3. **ROUND-1 spec failure mode**: ROUND-1 assumed all 7 pages were envelope-sortable without verifying backend controllers. Result: 2 of 7 pages had wrong pattern classification (work-types, stock-movements, storage-items are pseudo-envelopes, not envelopes). Implementation hard gate: AC-10 verify-before-write.
