ИСХОДНОЕ СОСТОЯНИЕ
═══════════════════════════════════════════════════════════════

1. **`backend/src/common/interceptors/audit.interceptor.ts` lines 76-89:**

```ts
return next.handle().pipe(
  tap(async (data: unknown) => {
    const idFromResponse = (data as { _id?: string; id?: string })?. _id ?? ...;
    await this.audit.log({ action: meta.action, ... });   // ← ТИХИЙ throw после response
  }),
);
```

**Проблема:**
- NestJS уже отправил HTTP response клиенту (200/201/etc).
- Async callback в `tap()` ВЫШЕЛ из NestJS-awaitable chain.
- Если `this.audit.log()` throws → **Unhandled Promise Rejection**.
- Node v20+ default: warning в stderr + process.exit(1) при `unhandledRejection === 'strict'`.
- Production: если в `audit.log()` есть БД-rejection (TTL connect, replica fail) → crash всего API.

2. **`backend/src/common/interceptors/user-context.interceptor.ts` lines 24-36:**

```ts
return new Observable((subscriber) => {
  userContext.run(value, () => {
    next.handle().subscribe({                // ← ручная переподписка
      next: (v) => subscriber.next(v),
      error: (e) => subscriber.error(e),
      complete: () => subscriber.complete(),
    });
  });
});
```

**Проблема:**
- Manual `new Observable` + ручной `subscribe()` нарушает RxJS-share semantics.
- `next.handle()` может быть subscribed multiple times (Subscriber type, не Single).
- Каждая новая подписка → повторный run pipeline (request handler, Mongoose queries).
- ALC `userContext.run()` scope может end до того как async chain завершится → `userContext.getStore() === undefined` в Mongoose pre-hook → audit `userId === undefined`.

3. **`backend/src/common/interceptors/logging.interceptor.ts`:**

```ts
return next.handle().pipe(
  tap(() => {                                  // ← нет catchError/finalize
    const ms = Date.now() - start;
    this.logger.log(`${req.method} ${req.url} → ${ms}ms`);
  }),
);
```

**Проблема:**
- `tap()` без `catchError` / `finalize` → 4xx и 5xx не логируются.
- "Customer claims POST /orders returned 400" → нет логов расследовать.

ЧТО ДЕЛАТЬ
═══════════════════════════════════════════════════════════════

**ШАГ 1: audit.interceptor.ts — `tap(async)` → `mergeMap` + `catchError`**

1.1. Заменить pattern:
```ts
// ДО:
return next.handle().pipe(
  tap(async (data) => { await this.audit.log(...); }),
);

// ПОСЛЕ:
return next.handle().pipe(
  mergeMap(async (data) => {
    try {
      const idFromResponse = (data as { _id?: string; id?: string })?._id ?? (... )?.id;
      const reqUser = (req as RequestWithUser).user;
      await this.audit.log({ ... });
      return data;  // pass through
    } catch (err) {
      // Log locally BUT DO NOT propagate — response already sent.
      this.logger.error(
        `Audit log failed for ${meta.action} ${meta.entityType}: ${(err as Error).message}`,
        (err as Error).stack,
      );
      return data;
    }
  }),
);
```

1.2. Import `mergeMap` из `rxjs/operators`.

1.3. Добавить private readonly `logger = new Logger(AuditInterceptor.name)`.

**ШАГ 2: user-context.interceptor.ts — manual Observable → `from()` + `firstValueFrom`**

2.1. Заменить `new Observable + .subscribe()` pattern:
```ts
// ДО:
return new Observable((subscriber) => {
  userContext.run(value, () => {
    next.handle().subscribe({
      next: (v) => subscriber.next(v),
      error: (e) => subscriber.error(e),
      complete: () => subscriber.complete(),
    });
  });
});

// ПОСЛЕ:
import { firstValueFrom, from } from 'rxjs';
return from(
  userContext.run(value, () => firstValueFrom(next.handle())),
);
```

2.2. `firstValueFrom` гарантирует SINGLE subscription semantics для IF-non-multicast Observable (стандарт Nest interceptors).

2.3. Алтернативно: использовать `defer(() => from(Promise.resolve()))` если есть sharing concerns — но для Nest interceptors `firstValueFrom` is canonical.

**ШАГ 3: logging.interceptor.ts — `tap` → `finalize`**

3.1. Добавить error logging:
```ts
import { catchError, finalize, throwError } from 'rxjs';
import { tap as rxTap } from 'rxjs/operators';

return next.handle().pipe(
  rxTap({  // success branch
    next: () => {
      const ms = Date.now() - start;
      this.logger.log(`${req.method} ${req.url} → 2xx (${ms}ms)`);
    },
  }),
  catchError((err) => {
    const ms = Date.now() - start;
    const status = (err as { status?: number })?.status ?? 500;
    this.logger.warn(`${req.method} ${req.url} → ${status} (${ms}ms)`);
    return throwError(() => err);
  }),
  finalize(() => {
    // optional: log total span for slow requests >1s
    const ms = Date.now() - start;
    if (ms > 1000) this.logger.warn(`SLOW REQUEST: ${req.method} ${req.url} (${ms}ms)`);
  }),
);
```
