═══════════════════════════════════════════════════════════════
ИСХОДНОЕ СОСТОЯНИЕ
═══════════════════════════════════════════════════════════════

1. **Дублированный error-banner markup** в 8+ list-page templates:
   ```html
   @if (error()) {
     <div role="alert" class="mb-6 border hairline border-destructive rounded-sm px-4 py-3 text-sm text-destructive">
       {{ error() }}
     </div>
   }
   ```
   
   Pages с этим markup: materials/products/contracts/work-types/products-detail/categories/etc — В 8+ местах. **Maintainability risk:** UX-критичная markup нельзя менять через 1 источник правды. Если завтра product решит изменить цвет — 8 файлов надо править.

2. **`shared/util/on-dialog-close-once.ts` callback type:**
   ```ts
   onDialogCloseOnce<TResult>(ref, injector, callback: (result: TResult) => void);
   ```
   В page-usage: `(confirmed: unknown) => { if (!confirmed) return; ... }`. **Без `=== true` guard** — тип `unknown` остаётся unknown во всём handler. Если handler делает что-то типизированное (например `if (typeof result === 'string')`), он должен вручную добавлять type narrowing каждый раз.

3. **`product-detail.page.ts:75` empty-route 404 fire:**
   ```ts
   protected readonly productRes = httpResource<{...}>(() => ({
     url: `${this.baseUrl}/products/${this.idString()}`,  // /products/ (404) на initialValue
   }));
   ```
   
   `idString` = `''` initially (paramMap defaults to no `id`). httpResource eagerly fires → 404. **Пользователь видит «404 Not Found»** пока route не зарезолвится. If shared route resolver fix — нет защиты.

4. **`work-types.page.ts:198` `as any` cast в sort accessor:**
   `(r as any)[k]` — TypeScript escape. Companion к TZ-116 Step 1, но typo здесь specifically.

ШАГ 1: Extract `<app-pi-error-banner>` shared component

1.1. Создать `frontend/src/app/shared/ui/error-banner/pi-error-banner.component.ts`:
```ts
@Component({
  selector: 'app-pi-error-banner',
  changeDetection: ChangeDetectionStrategy.OnPush,
  template: `
    @if (error()) {
      <div role="alert" class="pi-error-banner">
        {{ error() }}
      </div>
    }
  `,
  styles: [`
    .pi-error-banner {
      margin-bottom: 1.5rem;
      border: 1px solid var(--color-rule);
      border-color: var(--color-destructive);
      border-radius: 2px;
      padding: 12px 16px;
      font-size: 13px;
      color: var(--color-destructive);
    }
  `],
})
export class PiErrorBannerComponent {
  readonly error = input<string | null>(null);
}
```

1.2. **Adoption:** 8+ pages заменяют inline markup:
```html
<!-- BEFORE -->
@if (error()) {
  <div role="alert" class="mb-6 border hairline border-destructive rounded-sm px-4 py-3 text-sm text-destructive">
    {{ error() }}
  </div>
}

<!-- AFTER -->
<app-pi-error-banner [error]="error()" />
```

1.3. Barrel export: `shared/ui/error-banner/index.ts`.

1.4. **Anti-scope**: TZ-105.3 уже в плане делает похожее. **Если TZ-105.3 был реализован** — не дублировать; coordinate через STATUS.md.

ШАГ 2: Tighten `onDialogCloseOnce` type signature

2.1. `shared/util/on-dialog-close-once.ts`:
```ts
// CURRENT
type Confirmed = unknown;

// NEW: enforces boolean disambiguation
type ConfirmFn = (confirmed: boolean) => void;
type ResultFn<TResult> = (result: TResult) => void;

// Overloads:
onDialogCloseOnce(ref, injector, callback: ConfirmFn, defaultValue?: false): void;
onDialogCloseOnce<TResult>(ref, injector, callback: ResultFn<TResult>): void;
```

2.2. Update all page call sites:
```ts
// Before:
onDialogCloseOnce(ref, this.injector, (confirmed: unknown) => {
  if (!confirmed) return;
  // ...
});

// After:
onDialogCloseOnce(ref, this.injector, (confirmed: boolean) => {
  if (!confirmed) return;  // now type is boolean
  // ...
});
```

2.3. Page-level call sites to update: 7+ pages. Mechanical replacement.

ШАГ 3: Guard `product-detail.page.ts` from empty-route fire

3.1. Fix `protected readonly productRes`:
```ts
protected readonly productRes = httpResource<...>(() => {
  const id = this.idString();
  if (!id || id === 'null') return undefined;  // skip request
  return { url: `${this.baseUrl}/products/${id}` };
});
```

3.2. Add loading indicator for "ID not yet resolved":
```html
@if (!idString()) {
  <p class="text-sm text-muted-foreground">Загрузка идентификатора…</p>
}
```

3.3. Apply to **all** detail pages that paramMap-resolve: `module-detail.page.ts`, … (search for `idString() ?? ''` pattern + add guard).
