## Шаг 1: Security — удалить credentials из git дерева (CRITICAL) ⚠️

**Файлы:** `jwt.access`, `jwt.refresh` — валидные JWT-токены (admin/admin) висят в корне проекта. Если проект публичный или кто-то сделает `git push`, это компрометация.

### Что сделать:

1. **Добавить в `.gitignore`:**
   ```
   # JWT secrets — НЕ КОММИТИТЬ (TZ-142)
   jwt.access
   jwt.refresh
   ```

2. **Удалить из git (без потери локальных копий):**
   ```bash
   git rm --cached jwt.access jwt.refresh
   ```

3. **Заменить seed-токены на плейсхолдер** (если они используются в тестах/скриптах):
   - Создать `jwt.access.example` и `jwt.refresh.example` с токенами-заглушками
   - Добавить `jwt.access.example` и `jwt.refresh.example` в git
   - Адаптировать `start.mjs` и reset-password скрипты под чтение `.example` если нет реального файла

4. **Проверить references:**
   ```bash
   grep -r "jwt.access\|jwt.refresh" --include="*.ts" --include="*.js" --include="*.mjs"
   ```
