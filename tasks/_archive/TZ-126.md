1. **`backend/src/common/eav/eav.service.ts` resolveAttributes — lines 41-66:**

```ts
const result: Record<string, unknown> = {};
for (const [name, value] of Object.entries(raw)) {
  const def = defsByName.get(name);
  if (!def) { this.logger.warn(...); continue; }
  const parsed = this.validateAndParse(name, value, def);
  result[name] = parsed;
  await this.valueModel.updateOne(        // ← N sequential, NO TRANSACTION
    { entityType, entityId, attributeId: def._id },
    { $set: { value: parsed } },
    { upsert: true },
  ).exec();
}
return result;
```

**Проблема:**
- 10 атрибутов = 10 sequential `await`.
- MongoDB write-concern fail на 5-м (`replica fail`, `timeout`, `disk full`):
  - Атрибуты 1-4 уже в БД → permanent state.
  - Атрибуты 5-10 lost → return value говорит "saved" но БД частично.
- **NO atomicity.** Partial-write = data corruption.
- `return result` всегда resolves OK — caller (product.service или cart-item.service) не видит error.

2. **`backend/src/common/eav/eav.service.ts` validateAndParse — enum case (lines 119-126):**

```ts
case 'enum': {
  const s = String(value);
  if (!def.options || !def.options.includes(s)) {
    throw new BadRequestException(
      `Attribute "${name}" must be one of: ${def.options?.join(', ')}`,
    );
  }
  return s;
}
```

**Проблема:**
- `String(value)` НЕ trim → `"Yes "` (с trailing space) rejected vs `"Yes"` accepted.
- UI-формы часто submit с whitespace (autocomplete, copy-paste validation).
- У некторых операторов в seed-данных есть `def.options = ['Yes', 'No']` — нарушение может быть legitimate per spec.

3. **`backend/src/common/eav/eav.service.ts` setValue — single updateOne:**

- Меньший риск (1 op), но если FK constraint fail (attribute belongs to wrong entityType) — нет rollback, parent entity в inconsistent state.

**ШАГ 1: resolveAttributes — bulkWrite enforced в startSession().withTransaction()**

1.1. Refactor signature (no breaking — internal):
```ts
async resolveAttributes(
  entityType: string,
  entityId: Types.ObjectId,
  raw: Record<string, unknown>,
  categoryId?: Types.ObjectId,
): Promise<Record<string, unknown>> {
  if (!raw || Object.keys(raw).length === 0) return {};

  // 1. Pre-validation в одной фазе (все attrs valid? если нет — fail fast)
  const defFilter: Record<string, unknown> = { entityType, isActive: true };
  if (categoryId) {
    defFilter.$or = [{ categoryId }, { categoryId: null }, { categoryId: undefined }];
  }
  const defs = await this.defModel.find(defFilter).exec();
  const defsByName = new Map(defs.map((d) => [d.name, d]));

  // 2. Parse ВСЕХ attrs перед ANY write — fail-fast semantics
  const parsed: Record<string, { name: string; value: unknown; def: AttributeDefinitionDocument }> = {};
  for (const [name, value] of Object.entries(raw)) {
    const def = defsByName.get(name);
    if (!def) {
      this.logger.warn(`Unknown attribute "${name}" for ${entityType} ${entityId}`);
      continue;
    }
    parsed[name] = { name, value: this.validateAndParse(name, value, def), def };
  }

  if (Object.keys(parsed).length === 0) return {};

  // 3. bulkWrite в transaction (atomic)
  const ops = Object.values(parsed).map(({ def, value: val }) => ({
    updateOne: {
      filter: { entityType, entityId, attributeId: def._id },
      update: { $set: { value: val } },
      upsert: true,
    },
  }));

  const session = await this.valueModel.db.startSession();
  try {
    await session.withTransaction(async () => {
      // explicit `ordered: true` (Mongoose default) — на провале op N остальные не выполняются.
      // Идемпотентность достигается через session.withTransaction abort → весь bulkWrite rolled back.
      // НЕ использовать `ordered: false` — outside-of-transaction semantics «partial may survive»
      // и противоречит цели атомарности.
      await this.valueModel.bulkWrite(ops, { session });
    });
  } finally {
    await session.endSession();
  }

  return Object.fromEntries(
    Object.entries(parsed).map(([k, v]) => [k, v.value]),
  );
}
```

1.2. Требования к окружению:
- MongoDB replica set required (`withTransaction` НЕ работает на standalone). Если standalone — fallback explicit message.
- Уточнить: kppdf-8.0 в docker-compose использует replica set (см. `deploy/synology/config.env.example`).

**ШАГ 2: validateAndParse enum — explicit `.trim()` + case policy**

2.1. Определить explicit policy (без двусмысленности):
- Trim только для string enum (whitespace cosmetic).
- Не менять case (case-sensitive validation: "Yes" ≠ "yes").
- Trim ДО stringification:
```ts
case 'enum': {
  const s = String(value).trim();  // ← explicit trim
  if (!def.options || !def.options.includes(s)) {
    throw new BadRequestException(
      `Attribute "${name}" must be one of: ${def.options?.join(', ')} (got: "${s}")`,
    );
  }
  return s;
}
```

2.2. Применить тот же `.trim()` для `string` case (не теряем данные при input mutation, только normalizing), и для `date` — нормализация iso-string.

**ШАГ 3: setValue / deleteValue — НЕ требуют транзакции**

3.1. Оставить как есть. Эти методы — единичная операция, transaction overhead > benefit. Не делать рефактор ради рефактора.
