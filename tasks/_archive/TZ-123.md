**Причина появления:**
- В Schema поле `optional: true` (`@Prop({ required: false })`) типизировано как `Types.ObjectId | undefined`.
- В DTO поле опционально (`@IsOptional()`), типизировано как `string | undefined`.
- В service при `doc.field = dto.field ? new ObjectId(dto.field) : undefined` TS даёт ошибку: `Types.ObjectId | undefined` assignable to `Types.ObjectId`. Workaround — `as unknown as Types.ObjectId`.

**Проблемы:**
- Ломает intellisense (поле вроде ObjectId, но может быть undefined).
- Не видно реальной ошибки при изменении типа.
- Code reviewer видит hack → снижает trust.
- Удлиняет поддержку: рефактор схемы = проход по 12+ сервисам.

ЧТО ДЕЛАТЬ
═══════════════════════════════════════════════════════════════

**ШАГ 1: Создать `ToObjectId` decorator**

1.1. В `backend/src/common/decorators/to-object-id.decorator.ts`:
```ts
import { Transform } from 'class-transformer';
import { Types } from 'mongoose';

export const ToObjectId = () =>
  Transform(({ value }) =>
    value && typeof value === 'string' && Types.ObjectId.isValid(value)
      ? new Types.ObjectId(value)
      : value === ''
      ? undefined
      : value,
  );

// Variant для опциональных полей:
export const ToOptionalObjectId = () =>
  Transform(({ value }) => {
    if (value === null || value === undefined || value === '') return undefined;
    if (typeof value === 'string' && Types.ObjectId.isValid(value)) {
      return new Types.ObjectId(value);
    }
    return value;
  });
```

1.2. Зарегистрировать глобальный `plainToInstance` через `class-transformer` (если уже настроен — нет).

**ШАГ 2: Обновить DTOs (top-6 в scope)**

2.1. В каждом DTO, где есть `optional ObjectId field`, добавить `@ToOptionalObjectId()`:
```ts
// create-work-type.dto.ts
export class CreateWorkTypeDto {
  @IsString() name!: string;
  @IsString() section!: string;
  @IsOptional() @IsString() description?: string;
  @IsOptional() @ToOptionalObjectId() workCenterId?: Types.ObjectId;
  ...
}
```

2.2. В каждом Update DTO — то же самое.

**ШАГ 3: Refactor сервисов (top-6 в scope)**

3.1. Убрать `(undefined as unknown as Types.ObjectId)` в update:
```ts
// До:
if (dto.workCenterId !== undefined) {
  doc.workCenterId = dto.workCenterId
    ? new Types.ObjectId(dto.workCenterId)
    : (undefined as unknown as Types.ObjectId);
}

// После:
if (dto.workCenterId !== undefined) {
  doc.workCenterId = dto.workCenterId; // уже ObjectId после transform
}
```

3.2. Убрать `(x as unknown as Document)` casts в bom/cost-calculation — добавить explicit populate в findAll.

**ШАГ 4: Validation**

4.1. Создать `backend/src/common/validators/is-optional-object-id.validator.ts` если decorator недостаточен.

4.2. Включить в DTO `@IsOptional() @IsObjectId() @ToOptionalObjectId()` для всех ObjectId refs.

**ШАГ 5: Тесты + smoke**

5.1. Unit: `work-type.service.spec.ts` (если отсутствует — создать):
- Create с валидным `workCenterId` → doc.workCenterId is Types.ObjectId.
- Create с пустой строкой `workCenterId=""` → doc.workCenterId is undefined.

5.2. E2E: для одного endpoint (e.g. PUT /work-types/:id) — отправить с `workCenterId=""` → 200, без 400.
