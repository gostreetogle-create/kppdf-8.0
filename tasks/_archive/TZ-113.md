ИСХОДНОЕ СОСТОЯНИЕ
═══════════════════════════════════════════════════════════════

1. `frontend/src/app/pages/doc-constructor/builder/block-renderer.component.ts` `onSelect()`:

   ```ts
   protected onSelect(event: Event): void {
     event.stopPropagation();
     if (event instanceof MouseEvent && (event.ctrlKey || event.metaKey)) {
       this.multiSelect.emit(this.block());
     } else {
       this.select.emit(this.block());
     }
   }
   ```

   **Проблема A — a11y gap:**
   - Multi-select РАБОТАЕТ только через `MouseEvent`. Пользователи с keyboard / screen reader / mobile (touch events НЕ имеют ctrlKey) **не могут сделать multi-select**.
   - Стандартные паттерны в macOS/Linux: **Cmd/Ctrl+Click** для toggle select (как сейчас).
   - Нет keyboard эквивалента: ни **Shift+Click** (range select), ни **Ctrl+Space** (toggle), ни **Cmd+A** (select all in list), ни **Ctrl+Home/End** (select to start/end).

   **Проблема B — нет ARIA атрибутов:**
   - `<div cdkDrag>` не имеет `role`, `aria-selected`, `aria-checked`.
   - Screen reader озвучивает «block 1 button» — не информативно.

   **Проблема C — `[attr.aria-label]` без `tabindex`:**
   - Block не является focusable element (нет `tabindex`).
   - Keyboard user не может Tab'ом добраться до блока.

2. `frontend/src/app/pages/doc-constructor/builder/builder-canvas.component.ts`:
   - Контейнер `<div cdkDropList>` не имеет `role="listbox"` для keyboard navigation context.
   - Не обрабатывает keyboard events на уровне container.

ШАГ 1: Сделать блок focusable + ARIA-compliant

1.1. В `block-renderer.component.ts` template:

```html
<div
  cdkDrag
  tabindex="0"
  role="option"
  [attr.aria-selected]="selected()"
  [attr.aria-checked]="multiSelected()"
  [class.is-selected]="selected()"
  [class.is-multi-selected]="multiSelected()"
  ...
  (click)="onSelect($event)"
  (keydown)="onKeyDown($event)"
>
```

1.2. `aria-selected` для single-selection mode, `aria-checked` для multi-selection mode (semantic checkbox-like).

1.3. Добавить tabindex management: только selected|multiSelected|prevSelected получают `tabindex="0"`, остальные `tabindex="-1"` (для roving tabindex pattern).

- Имплементация: блок вычисляет свой tabindex:
  ```ts
  readonly tabIndex = computed<'0' | '-1'>(() => {
    if (this.selected() || this.multiSelected()) return '0';
    // Иначе roving — selected block всегда tabbable
    return '-1';
  });
  ```
- На уровне canvas: single focusable block (selected). При arrow-keys → перемещение focus = selection.

ШАГ 2: Keyboard handler

2.1. В `block-renderer.component.ts`:

```ts
protected onKeyDown(event: KeyboardEvent): void {
  const k = event.key;
  // Single select
  if (k === 'Enter' || k === ' ') {
    event.preventDefault();
    this.select.emit(this.block());
    return;
  }
  // Multi-select toggle
  if ((k === ' ' && (event.ctrlKey || event.metaKey)) || k === 'Insert') {
    event.preventDefault();
    this.multiSelect.emit(this.block());
    return;
  }
  // Range select (Shift+Enter)
  if (k === 'Enter' && event.shiftKey) {
    event.preventDefault();
    this.rangeSelect.emit(this.block());  // НОВЫЙ OUTPUT — для canvas обрабатывает
    return;
  }
  // Remove from selection
  if (k === 'Delete' || k === 'Backspace') {
    // Только если в multi-selected state
    if (this.multiSelected()) {
      event.preventDefault();
      // Canvas обрабатывает удаление (TZ-111)
      this.delete.emit(this.block());
    }
    return;
  }
}
```

2.2. WAI-ARIA pattern: keyboard user ⇄ mouse user должна быть эквивалентная функциональность. Tab → focus блок → Enter/Space = single select. Shift+Enter = range add. Ctrl+Space = multi-toggle.

ШАГ 3: Container-level navigation

3.1. В `builder-canvas.component.ts`:

```ts
readonly rangeSelect = output<TemplateBlock>();
protected onRangeSelect(block: TemplateBlock): void {

```

3.2. Builder page (или BuilderCanvas) обрабатывает range select:

```ts
protected onRangeSelect(block: TemplateBlock): void {
  // Получить index текущего selected/focused и index этого блока
  // Все между ними → добавить в selectedIds
  const endKey = blockKey(block);
  const endIdx = this.blocks().findIndex(b => blockKey(b) === endKey);
  
  // Last anchor — последний single-selected, или первый multi-selected
  const anchor = this.lastSingleSelected() ?? this.firstMultiSelected();
  if (!anchor) return;
  
  const startIdx = this.blocks().findIndex(b => blockKey(b) === anchor);
  
  const [lo, hi] = startIdx < endIdx ? [startIdx, endIdx] : [endIdx, startIdx];
  const rangeIds = new Set<string>();
  for (let i = lo; i <= hi; i++) {
    rangeIds.add(blockKey(this.blocks()[i]));
  }
  
  this.selectedIds.set(new Set([...this.selectedIds(), ...rangeIds]));
  this.selectedId.set(null);
}
```

ШАГ 4: Container role + true roving tabindex navigation (WAI-ARIA listbox)

4.1. Builder canvas template:

```html
<div
  cdkDropList
  role="listbox"
  aria-label="Блоки шаблона документа"
  aria-multiselectable="true"
  [id]="CANVAS_DROPLIST_ID"
  [cdkDropListData]="blocks()"
  (cdkDropListDropped)="onDrop($event)"
>
```

4.2. **True roving tabindex pattern** (НЕ group-tabbing, как указано ранее в section??):
- ТОЛЬКО ОДИН child в любой момент имеет `tabindex="0"`. Это «focusable element» container-а.
- Все остальные children имеют `tabindex="-1"`.
- Управление в canvas (Container owner): signal `focusedBlockIndex` → canvas рендерит `tabindex` каждому ребёнку через `block.focusable = (i === focusedBlockIndex)`.
- **Mouse click** → focusedBlockIndex ← clicked index (canvas service-side).
- **ArrowUp/ArrowDown keyboard** → increment/decrement focusedBlockIndex → первый блок после change получает `tabindex="0"` и получает `.focus()` (ElementRef.focus).
- **Home/End** → focusedBlockIndex ← 0 / last.
- This matches https://www.w3.org/WAI/ARIA/apg/patterns/listbox/ (canonical listbox pattern).

4.3. Shift+Click (mouse) range-select — добавлен в § ШАГ 2 как keyboard; mouse эквивалент:
```ts
// в onSelect(event):
if (event instanceof MouseEvent) {
  if (event.shiftKey) {
    this.rangeSelect.emit(this.block());
  } else if (event.ctrlKey || event.metaKey) {
    this.multiSelect.emit(this.block());
  } else {
    this.select.emit(this.block());
  }
}
```

4.4. Удалить из § ШАГ 1 «tabindex management» — он относится к **group-tabbing**, не roving. Tabindex будет приходить из canvas через input/property `focusable: boolean` на block.
