   - materials.page.ts:344 — `protected reload(): void { this.listRes.reload(); }` — **method exists, но НЕТ кнопки в template**.

4. **`pi-input` `type="search"` без default browser behavior:**
   - В большинстве браузеров native `type="search"` имеет встроенную ❌-кнопку. **Но** эта кнопка native и не программируется (не вызывает наш search input handler).
   - Custom UI reset button лучше для consistency.

ШАГ 2: Custom reset (clear search button)

2.1. Создать новый shared `<app-pi-search-input>` component:
```ts
@Component({
  selector: 'app-pi-search-input',
  template: `
    <div class="pi-search-wrap">
      <input
        type="search"
        [value]="value()"
        (input)="onInput($event)"
        [placeholder]="placeholder()"
        [attr.aria-label]="ariaLabel()"
        class="pi-input"
      />
      @if (value()) {
        <button
          type="button"
          class="pi-search-clear"
          (click)="onClear()"
          aria-label="Очистить"
        >×</button>
      }
    </div>
  `,
})
export class PiSearchInputComponent {
  readonly value = input.required<string>();
  readonly placeholder = input<string>('Поиск…');
  readonly ariaLabel = input<string>('Поиск');
  readonly valueChange = output<string>();
  readonly clear = output<void>();
}
```

2.2. Использовать во всех list-pages вместо raw `<input type="search">`.

ШАГ 3: Table row keyboard activation

3.1. Для pages с detail navigation (products, orders, contracts — TZ-83 план):
```html
<tr
  tabindex="0"
  role="link"
  [attr.aria-label]="'Открыть ' + row.name"
  (click)="openRow(row)"
  (keydown.enter)="openRow(row)"
  (keydown.space)="$event.preventDefault(); openRow(row)"
>
```

3.2. Для pages без detail (materials, work-types) — tabindex/role НЕ ставить; activate = focus first row-action button.

3.3. Если pi-table УЖЕ имеет row keyboard handler — расширить его вместо page-level.
