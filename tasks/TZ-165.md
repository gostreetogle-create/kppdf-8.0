## Проблема

После исправления layout-бага в `table-template-dialog.component.ts` (где `.ttd-body` с `height: 100%` затирал секцию preview, исправлено на `flex: 1; min-height: 0`), требуется **проверить все остальные form-dialog компоненты** на такие же баги.

### Ключевое наблюдение

**Ни один другой диалог не имеет ТОЧНО ТАКОГО ЖЕ бага**, как `table-template-dialog`, потому что:

1. **Только table-template-dialog** имеет two-section layout (список колонок + preview внизу) с кастомной flex-структурой (`.ttd-columns` + `.ttd-preview`)
2. **Только table-template-dialog** использует `::ng-deep` для установки `height: min(calc(100vh - 48px), 88vh)` на внешнем CDK overlay
3. **Только table-template-dialog** имеет `.ttd-body` с `height: 100%` вне pi-dialog body

Остальные form-dialog используют стандартный `<form body>` внутри `<app-pi-dialog>` — это простая вертикальная стопка секций.

### Общая проблема pi-dialog form variant

`pi-dialog.component.ts` для `variant="form"` задаёт body:

```ts
if (v === 'form') return 'px-6 py-6 text-sm text-ink';
// vs content:
if (v === 'content') return 'px-6 py-6 text-sm text-ink overflow-y-auto';
```

То есть у form variant **нет `overflow-y: auto`**. Внешний CDK overlay имеет `overflow-y: clip !important; max-height: 90vh`. Это означает:

- Если форма с FormArray вырастает выше ~90vh, **контент обрезается**
- Потенциально: 5+ элементов в FormArray + шапка формы = может превысить 90vh

На практике для текущих форм риск низкий — но это архитектурное ограничение.

## Requirements

### 1. (отменено — слишком глобально) Добавить overflow-y: auto только в 5 диалогов с FormArray

> ❌ **Отменено:** Добавление `overflow-y: auto` в pi-dialog form variant body глобально для всех 8 form-dialog
> избыточно — 3 из них (work-type, category, organization) — короткие формы, никогда не превысят 90vh.
> Риск: неожиданные артефакты (двойной скролл, смещение layout при появлении scrollbar).

✅ **Решение:** Добавить `overflow-y: auto` только в 5 диалогов, где есть растущий FormArray.

Изменить `<form body class="space-y-form-field">` на `<form body class="space-y-form-field overflow-y-auto min-h-0">`
в следующих компонентах:

| # | Файл | FormArray |
|---|------|-----------|
| 1 | `module-form-dialog.component.ts` | workTypes[] |
| 2 | `module-materials-form-dialog.component.ts` | materials[] |
| 3 | `order-form-dialog.component.ts` | items[] |
| 4 | `contract-form-dialog.component.ts` | items[] |
| 5 | `material-form-dialog.component.ts` | dimensions[] + photos |

**Почему это безопаснее:**
- Изменения изолированы — не затрагивают ~3 коротких диалога, которые работают корректно
- `overflow-y: auto` показывает scrollbar только когда контент превышает контейнер
- `min-height: 0` позволяет flex-item сжиматься ниже контента (prevent flex stretching)
- `<form body>` — проекция в `<ng-content select="[body]">` pi-dialog, scroll контейнер внутри body div

### 2. Добавить min-height: 0 в form body для всех form-dialog с FormArray

Для диалогов с FormArray (module-form, order-form, contract-form, material-form, module-materials-form) добавить `class="space-y-form-field min-h-0"` к `<form body>`.

Мотивация: `min-height: 0` в flex-контейнере позволяет flex-item'у сжиматься ниже контента — предотвращает растягивание формы за пределы контейнера при scroll overflow.

### 3. Проверить визуально

- Открыть каждый диалог в самом высоком сценарии (максимум элементов в FormArray)
- Убедиться, что:
  - footer (кнопки) всегда виден
  - контент скроллится внутри, а не за пределами overlay
  - нет пустых пространств

