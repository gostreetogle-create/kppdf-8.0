# TZ-157: Мониторинг — Sentry + Health Check + Uptime

**Priority**: HIGH  
**Dependencies**: —  
**Estimate**: 3-4h

## Проблема

На проде нет мониторинга:
- Нет Sentry (или аналога) для отслеживания ошибок
- Нет health check endpoint для Docker/k8s
- Нет uptime мониторинга
- Ошибки на проде остаются незамеченными

## Требования

1. **Backend: Health Check**
   - Установить `@nestjs/terminus`
   - Создать `HealthController` проверки:
     - MongoDB connection (MongooseHealthIndicator)
     - Disk space
     - Memory usage
   - Эндпоинт: `GET /api/health`

2. **Backend: Error Tracking**
   - Установить `@sentry/node`
   - Интегрировать в `HttpExceptionFilter` (отправка ошибок 500)
   - Настроить DSN через переменную окружения `SENTRY_DSN`
   - Source maps для прода

3. **Frontend: Error Tracking**
   - Установить `@sentry/angular`
   - Интегрировать в `ErrorHandler`
   - Настроить performance tracing

4. **Deploy: Uptime Monitoring**
   - Документировать настройку uptime-монитора (например, UptimeRobot / BetterStack)
   - Эндпоинт: `GET /api/health` для внешнего монитора

5. **Документация**
   - Обновить `deploy/config.env.example` с новыми переменными
   - Написать README по настройке мониторинга

## Acceptance Criteria

- [ ] `GET /api/health` возвращает `{ status: 'ok', info: {...} }`
- [ ] `GET /api/health` при отключённой MongoDB → `{ status: 'error' }`
- [ ] Sentry SDK установлен на backend и frontend
- [ ] Ошибки 500 попадают в Sentry
- [ ] `SENTRY_DSN` в `config.env.example`
