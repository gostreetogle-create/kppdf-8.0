# TZ-150: ESLint — добавить линтер на фронтенд

**Priority**: CRITICAL  
**Dependencies**: —  
**Estimate**: 2-3h

## Проблема

На фронтенде **нет ESLint** ни в каком виде:
- Нет `.eslintrc.*` или `eslint.config.*`
- Нет `lint` скрипта в `package.json`
- Нет pre-commit проверок
- Код может содержать неиспользуемые импорты, any, неправильные паттерны без контроля

## Требования

1. **Установить** `eslint` + `@typescript-eslint/parser` + `@typescript-eslint/eslint-plugin`
2. **Установить** `angular-eslint` (`@angular-eslint/eslint-plugin`, `@angular-eslint/template-parser`)
3. **Создать** `eslint.config.js` (flat config для ESLint v9+) с правилами:
   - `@typescript-eslint/no-unused-vars` — error
   - `@typescript-eslint/no-explicit-any` — warn (постепенно убирать)
   - `@angular-eslint/component-selector` — error (kebab-case)
   - `@angular-eslint/use-lifecycle-interface` — error
   - `@angular-eslint/prefer-standalone` — error (все компоненты standalone)
   - `@angular-eslint/no-output-on-prefix` — error
   - `no-console` — warn (кроме `Logger`)
   - `prefer-const` — error
4. **Добавить скрипты** в `frontend/package.json`:
   - `"lint": "eslint src/"` — проверка
   - `"lint:fix": "eslint src/ --fix"` — автоисправление
5. **Первый прогон**: запустить `pnpm run lint` и исправить все ошибки
6. **Интегрировать** с CI (см. TZ-151)

## Acceptance Criteria

- [ ] `pnpm run lint` выполняется без ошибок
- [ ] `pnpm run lint:fix` работает
- [ ] Все компоненты проходят проверки `@angular-eslint`
- [ ] Нет `any` без явного разрешения
- [ ] Код соответствует Angular style guide
