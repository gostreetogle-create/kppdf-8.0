3. **`PI_DIALOG_DATA` token (not PI_DIALOG_REF.data)** — DialogRef interface has no `.data` (only `.closed` + `.close()`). Dialog data lives in a separate `PI_DIALOG_DATA` token. (Applied in Phase C; reaffirmed in this review.)
4. **`CdkDropListDropped` → `CdkDragDrop<TemplateBlock[]>`** — `CdkDropListDropped` is NOT exported by `@angular/cdk/drag-drop` (the actual class is `CdkDragDrop<T>`). The grep error suggested `CdkDropListGroup` as a related but different type.
5. **CSS backticks in styles template literal** — the comment `/* parent group-hover (Tailwind \`group\` class) */` had backticks INSIDE a template literal, which closes the literal prematurely. TS1005: ',' expected. Fix: replace backticks with double quotes.
6. **extractErrorMessage runtime null guards** — `as HttpErrorResponse` cast (no `| null`) made typecheck pass but could crash at runtime if `httpResource.error()` returned null. Fix: explicit null guard `const err = ... as HttpErrorResponse | null; return err ? extractErrorMessage(err) : ''`.
7. **handleSaveResult + onAddBlock early-return on !res.ok** — the `if (res.ok) { ... res.data._id ... } else { res.error.status }` pattern failed TS2339 because `res.data` access in the success branch prevented the else branch from narrowing to `{ok: false, error: HttpErrorResponse}`. Fix: refactor to early-return on `!res.ok` (canonical TS pattern for discriminated unions).
8. **Minor cleanups**: `_BlockTypeRef` anti-pattern removed, `parts: string[]` annotation in block-renderer binding-badge, `FileText` import added to app-layout (was missing for 4th NAV_CATEGORY), `Subscription` unused field/import removed, `takeUntilDestroyed` covers cleanup.
