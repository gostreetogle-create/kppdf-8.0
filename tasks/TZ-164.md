# TZ-164: Husky + lint-staged — pre-commit проверки

**Priority**: LOW  
**Dependencies**: TZ-150 (ESLint), TZ-153 (Prettier)  
**Estimate**: 1-2h

## Проблема

Нет pre-commit hooks:
- Можно закоммитить код с ESLint ошибками
- Можно закоммитить неформатированный код
- Тесты не прогоняются перед коммитом
- Нарушения code style попадают в репозиторий

## Требования

1. **Установить** `husky`:
   ```bash
   pnpm add -D husky
   npx husky init
   ```
2. **Настроить** `.husky/pre-commit`:
   ```bash
   #!/usr/bin/env sh
   pnpm run lint-staged
   ```
3. **Установить** `lint-staged`:
   ```bash
   pnpm add -D lint-staged
   ```
4. **Добавить** в `package.json` (корневой или frontend):
   ```json
   "lint-staged": {
     "*.ts": ["eslint --fix", "prettier --write"],
     "*.html": ["prettier --write --parser angular"],
     "*.{scss,css}": ["prettier --write"]
   }
   ```
5. **Установить** `@commitlint/config-conventional` (опционально):
   - Проверка формата commit messages
   - `type(scope): description`

## Acceptance Criteria

- [ ] `git commit` автоматически запускает lint-staged
- [ ] ESLint + Prettier фиксят файлы перед коммитом
- [ ] Можно пропустить hooks через `--no-verify` (emergency bypass)
- [ ] (Опционально) commitlint проверяет формат сообщений
