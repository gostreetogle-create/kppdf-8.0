# TZ-161: Lighthouse CI — Performance Regression Tracking

**Priority**: MEDIUM  
**Dependencies**: TZ-151 (CI/CD)  
**Estimate**: 2-3h

## Проблема

Нет автоматического Performance Budget enforcement:
- Нельзя гарантировать, что новый код не ухудшит производительность
- Нет истории производительности (регрессии)
- Lighthouse проверки только ручные

## Требования

1. **Установить** `@lhci/cli`:
   ```bash
   pnpm add -D @lhci/cli
   ```
2. **Создать** `lighthouserc.js`:
   ```js
   module.exports = {
     ci: {
       collect: {
         url: [
           'http://localhost:4200/',
           'http://localhost:4200/materials',
           'http://localhost:4200/orders',
           'http://localhost:4200/products',
         ],
         numberOfRuns: 3,
       },
       assert: {
         assertions: {
           'categories:performance': ['error', { minScore: 0.8 }],
           'categories:accessibility': ['error', { minScore: 0.85 }],
           'categories:best-practices': ['error', { minScore: 0.9 }],
         },
       },
       upload: {
         target: 'temporary-public-storage',
       },
     },
   };
   ```
3. **Добавить** скрипт в `frontend/package.json`:
   ```json
   "lighthouse": "lhci autorun"
   ```
4. **Добавить** в CI:
   - Build → serve → `pnpm run lighthouse`
   - Non-blocking для PR (только предупреждение)
   - Блокирующий для main

## Acceptance Criteria

- [ ] `lhci autorun` собирает метрики для 4 страниц
- [ ] Performance threshold: 80+
- [ ] Accessibility threshold: 85+
- [ ] CI падает при нарушении бюджетов на main
