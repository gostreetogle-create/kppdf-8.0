# Аудит и рефакторинг секции «Декорации» — фоновые изображения

## Проблемы текущей реализации
1. Фоновые изображения просто стопятся друг на друга (`position: absolute; inset: 0; opacity: 0.4`) — непонятно зачем несколько
2. Нет привязки к страницам — все фоны отображаются одновременно
3. Нет понятия «фон по умолчанию»
4. Изображения масштабируются через `object-fit: cover` — обрезаются, не вписываются в А4
5. Нет управления прозрачностью через UI (поле есть в схеме, но нет слайдера)
6. Нет удаления отдельных фонов
7. Backend HTML генерация: `position: fixed; inset: 0` — не привязана к странице
8. Нет поддержки альбомной ориентации
9. Canvas opacity захардкожен 0.4, не читает `backgroundOpacity` из шаблона

## Решение

### 1. Новая модель данных для фонов

**Schema** (`document-template.schema.ts`):
```typescript
// Заменить backgroundImage: string[] на:
@Prop({ type: [{
  url: String,
  isDefault: { type: Boolean, default: false },
  pageIndex: { type: Number, default: null },  // null = назначается позже
}], default: [] })
backgrounds!: Array<{ url: string; isDefault: boolean; pageIndex: number | null }>;

// Добавить:
@Prop({ default: 'portrait' })
orientation!: 'portrait' | 'landscape';
```

**Миграция**: метод `migrateBackgroundImages()` в сервисе — при первом чтении шаблона со старым форматом `backgroundImage: string[]` конвертирует в новый формат.

### 2. Backend — upload endpoint

**Изменить** `uploadBackground()` в `document-template.service.ts`:
- При загрузке: если это первый фон — автоматически ставить `isDefault: true`
- Возвращать новый формат `backgrounds[]`

**Добавить** `DELETE /document-templates/:id/backgrounds/:index`:
- Удаляет фон по индексу из массива + удаляет файл с диска
- Если удаляемый фон был `isDefault` — следующий становится default

**Добавить** `PATCH /document-templates/:id/backgrounds/default`:
- Body: `{ index: number }` — переключает `isDefault` флаг

### 3. Backend — renderHtml()

**Изменить** CSS генерацию:
```css
.doc-page { 
  width: 210mm; min-height: 297mm; /* portrait */
  /* или width: 297mm; min-height: 210mm; /* landscape */
  position: relative; page-break-after: always; 
}
.doc-bg { 
  position: absolute; inset: 0; 
  background-size: contain; background-position: center; background-repeat: no-repeat;
  opacity: ...;
}
```

**Логика**: для каждой страницы找出 `isDefault` фон или конкретный `pageIndex` фон.

### 4. Frontend — Builder Canvas

**Изменить** `builder-canvas.component.ts`:
- Принимать `orientation` input
- Менять пропорции canvas в зависимости от ориентации
- Использовать `background-size: contain` вместо `cover` для корректного вписывания в А4
- Читать `backgroundOpacity` из пропсов, не хардкодить

### 5. Frontend — Builder Tool Pane (Декорации)

**Полностью переделать** секцию в `builder-tool-pane.component.ts`:

```
ДЕКОРАЦИИ
├── [список загруженных фонов]
│   ├── [миниатюра] Название_файла.png  [★ по умолчанию] [×]
│   ├── [миниатюра] другой_фон.jpg      [☆]              [×]
│   └── ...
├── + Загрузить фон
├── Прозрачность: ──●────── 40%
└── Ориентация: [Книжная] [Альбомная]
```

Каждый элемент списка:
- Миниатюра ( thumbnails 48×48)
- Имя файла
- Кнопка ★/☆ для установки default
- Кнопка × для удаления

### 6. Frontend — Builder Page

**Добавить** новые сигналы и обработчики:
- `onSetDefaultBackground(index)` — переключает default
- `onRemoveBackground(index)` — удаляет фон
- `onOrientationChange(orientation)` — меняет ориентацию

### 7. Frontend — DocumentTemplatesService

**Добавить** методы:
- `removeBackground(templateId, index)` → DELETE
- `setDefaultBackground(templateId, index)` → PATCH
- `updateOrientation(templateId, orientation)` → PATCH

## Верификация
1. Загрузить 2+ фона — видеть список с миниатюрами
2. Установить default — звездочка переключается
3. Удалить фон — удаляется из списка и с диска
4. Сменить ориентацию — canvas меняет пропорции
5. Прозрачность — слайдер влияет на preview и генерируемый HTML
6. Генерация HTML (`/build/:id`) — фон вписывается в А4, correct opacity
