          .dark on every change-detection cycle, undoing forceLightMode.

      3)  forceLightMode(page) → evaluate removeClass('dark').
  · Pass 'html' as axe.run context; rule tags in options.runOnly.
    Earlier rounds passed tags as context and got silent "No
    elements found" because axe treats arg-1 as CSS selectors.
  · Tagged RouteResult.error?: string; gate treats errors as
    blockers so a thrown axe.run never disguises as 0 violations.
  · Per-route summary + JSON dump + exit-code ladder (0 = pass,
    1 = blocker or error, 2 = script error).
