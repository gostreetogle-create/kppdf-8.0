ЧТО ДЕЛАТЬ
═══════════════════════════════════════════════════════════════

ШАГ 1: ProductionOrder (с транзакционным созданием OrderTask)

**Создание (`POST /production-orders`):**
- Вход: `{productId, quantity, plannedStartDate, plannedEndDate, ralCode?, workCenterId?, contractId?, proposalId?}`.
- **Транзакция (session.withTransaction):**
  1. Сгенерировать `number` через `Counter.next('ProductionOrder', 'PO')`.
  2. Создать `ProductionOrder` документ.
  3. Загрузить `TechProcess` (active) + `Bom` (active) для `productId`.
  4. Для каждой операции из `TechProcess.operations` — создать `OrderTask` с `productionOrderId`, `workTypeId`, `workCenterId`, `estimatedHours`, `plannedStartDate` (с учётом cumulative duration), `dependsOnTaskIds` (предыдущая задача).
  5. Для каждого component из `Bom.components` — создать `OrderTask` с `componentId`, `material` (если есть в БД).
  6. Обновить `ProductionOrder.status` → 'planned'.
- `GET /production-orders` с populate: tasks, product, contract, proposal.
- `GET /production-orders/:id/tasks` — все задачи с зависимостями.
- `POST /production-orders/:id/start` — переводит в статус 'in_progress' (через StatusWorkflow из TZ-05), `actualStart = new Date()`.
- `POST /production-orders/:id/close` — OrderClosing flow (см. ШАГ 7).

ШАГ 2: OrderTask (CRUD + зависимости)

- `GET /order-tasks?productionOrderId=...&status=...`.
- `PATCH /order-tasks/:id` — обновить status, actualHours, actualStartDate/End.
- `POST /order-tasks/:id/complete` — отметить выполненной, обновить `actualEndDate`, проверить что все `dependsOnTaskIds` тоже complete.
- `GET /order-tasks/ready?workerId=...` — задачи готовые к выполнению (зависимости выполнены, worker назначен).

ШАГ 3: WorkOrder (наряд)

- Создаётся на основе OrderTask (через `POST /order-tasks/:id/work-order {qty, assignedTo, startDate, endDate}`).
- При создании — генерирует `WorkOrderOperation[]` на основе WorkType.
- `GET /work-orders/:id` с populate operations, order, product, assignedTo.
- `POST /work-orders/:id/complete` — отмечает все операции как done.

ШАГ 4: WorkOrderOperation (CRUD)

- `POST /work-order-operations/:id/start` — `startedAt = new Date()`.
- `POST /work-order-operations/:id/complete {actualDuration}` — `completedAt`, `actualDuration`.

ШАГ 5: CostCalculation (плановая себестоимость)

- `POST /products/:id/cost-calculations` — рассчитать из активного Bom:
  - `totalMaterialCost` = sum(material.pricePerUnit × bom.components[i].quantity)
  - `totalLaborCost` = sum(workType.hourlyRate × techProcess.operations[i].durationHours)
  - `overheadCost` = totalMaterialCost × overheadPercent / 100
  - `totalCost` = totalMaterialCost + totalLaborCost + overheadCost
- `GET /products/:id/cost-calculations?isActive=true` — последняя активная.
- `POST /cost-calculations/:id/activate`.

ШАГ 6: ActualCost (фактические затраты)

- `POST /production-orders/:id/actual-costs` — `{type, amount, description, sourceRef, date}`.
- `GET /production-orders/:id/actual-costs` — список.
- `GET /production-orders/:id/cost-comparison` — `{planned: CostCalculation.totalCost, actual: sum(ActualCost.amount), variance: actual - planned}`.

ШАГ 7: OrderClosing (закрытие заказа)

- `POST /production-orders/:id/close {closingType, number, date, amount, organizationId, fileUrl?}`.
- Проверить: все OrderTask → status='completed' (через StatusWorkflow).
- Создать `OrderClosing` документ.
- `GET /production-orders/:id/closing` — получить.
