# example_tz_filled — Заполненный пример реального TZ
# (для ориентира при создании первого TZ-NN.txt)

> Этот файл — **наглядный пример**, как должен выглядеть готовый TZ-NN.txt
> после заполнения из [TZ-template.txt](TZ-template.txt). Не копируйте
> дословно — адаптируйте под свой проект. Файл НЕ начинается с `TZ-`,
> чтобы verify-status.sh не считал его за настоящее TZ.

---

═══════════════════════════════════════════════════════════════
TZ-01: Пример CSS-токенов для светлой темы
═══════════════════════════════════════════════════════════════

РОЛЬ АГЕНТА: Frontend CSS Architect

ЗАВИСИМОСТИ: TZ-00 (Foundation: design tokens + CSS architecture) —
             должен быть DONE перед стартом этого TZ. Если TZ-00 ещё
             в работе, отложите как `_archive/<YYYY-MM>/TZ-01.deferred.txt`.

LAYER: 1   ← ОБЯЗАТЕЛЬНО (см. правила параллельности в TZF-00.txt)

CONFLICT KEYS: ← ОБЯЗАТЕЛЬНО
  - frontend/src/styles.css
  - frontend/src/app/styles/tokens.css

═══════════════════════════════════════════════════════════════
ИСХОДНОЕ СОСТОЯНИЕ
═══════════════════════════════════════════════════════════════

1. Стили приложения находятся в `frontend/src/styles.css`.
   В нём определены CSS custom properties на `:root` (текущая дефолтная)
   и `[data-theme='dark']` (тёмная тема).

2. Проблемы:
   - Светлая тема не вынесена в отдельный блок `[data-theme='light']`,
     поэтому переключатель темы «застревает» на дефолтной.
   - Контраст текста по умолчанию в светлой теме — 3.2:1
     (WCAG AA требует 4.5:1).

═══════════════════════════════════════════════════════════════
ЧТО ДЕЛАТЬ
═══════════════════════════════════════════════════════════════

ШАГ 1: Добавить блок `[data-theme='light']` в `frontend/src/styles.css`.
   Скопировать все переменные из `:root` (кроме `--bg-primary` и company-color
   accent). Для каждой цветовой переменной выбрать значение, дающее контраст
   ≥ 4.5:1 с белым фоном `#FFFFFF`.

ШАГ 2: Обновить `--text-primary` для светлой темы.
   Сейчас `#1a1a1a` → на `#1F2937` (slate-800) для контраста 12.6:1.

ШАГ 3: Добавить theme-switcher в `frontend/src/main.ts`.
   Прочитать localStorage `app-theme` → применить `data-theme`
   атрибут к `<html>` (если есть), иначе оставить дефолтную.

ШАГ 4: Smoke-проверка вручную.
   Открыть http://localhost:4200, нажать «Toggle Theme», убедиться:
   - body bg меняется с тёмного на светлый;
   - текст читается без напряжения;
   - все кнопки остаются кликабельными.

═══════════════════════════════════════════════════════════════
ФАЙЛЫ ДЛЯ ИЗМЕНЕНИЯ
═══════════════════════════════════════════════════════════════

ИЗМЕНЯТЬ:
- frontend/src/styles.css    [добавить блок `[data-theme='light']`]
- frontend/src/main.ts        [добавить theme-switcher через localStorage]

НЕ ИЗМЕНЯТЬ (явно):
- styles/tokens.css           [общие токены — менять не нужно]
- TS/HTML-файлы компонентов [стилизуются через переменные]
- _templates/*                [служебные файлы kit-а]

═══════════════════════════════════════════════════════════════
КРИТЕРИИ ПРИЁМКИ
═══════════════════════════════════════════════════════════════

1. `bash verify-status.sh` возвращает `PASS`.
2. После переключения темы в localStorage: `data-theme="light"` в DOM,
   `--bg-primary` стал светлым (#FFFFFF или близко).
3. Текст body имеет контраст ≥ 4.5:1 (проверить через DevTools →
   Accessibility → Contrast Ratio).
4. `cd frontend && npm run build` проходит без ошибок.
5. Ручная проверка: открыть приложение, переключать тему 10 раз —
   UI не «мигает», нет JS-ошибок в консоли.

═══════════════════════════════════════════════════════════════
КОНФЛИКТ-ЧЕК-ЛИСТ (для параллельных запусков)
═══════════════════════════════════════════════════════════════

Перед стартом ШАГ 0 прочитать содержимое `_active/`:
- Если там TZ, который тоже трогает `frontend/src/styles.css` —
  → ОСТАНОВИТЬСЯ, скопировать себя в
    `_archive/<YYYY-MM>/TZ-01.deferred.txt` (DEFERRED-маркер,
    формат в TZF-00.txt) и сообщить PO.
- Если в `_active/` ничего — ОК, начинать ШАГ 0.

═══════════════════════════════════════════════════════════════
ПОДСКАЗКИ ДЛЯ PO (как адаптировать этот пример)
═══════════════════════════════════════════════════════════════

1. Замените номер `01` на свой (`02`, `03`...) — файл переименуется
   в `TZ-02.txt`, `TZ-03.txt` и т.д.
2. Замените роли/зависимости/файлы под свой стек.
3. Критериев приёмки — 3-5, не больше. Они должны быть ИЗМЕРИМЫМИ.
4. Этот файл-пример НАЗОВИТЕ иначе (например, в честь своей первой
   фичи), чтобы имя не совпадало с реальным TZ-NN.
