**Major debugging arc (recorded for archival):**

ROOT CAUSE на этапе typecheck занимал 4 iterations:
1. **Path depth**: 4 NEW files at `frontend/src/app/pages/doc-constructor/{texts,tables}/` are
   3 levels deep, но первый импорт использовал `../../shared/` (2 levels, OK для work-types).
   Все 4 файла нуждались в `../../../shared/` + `../../../core/`. Work-types 2-level.

2. **`httpResource` at bottom of file**: `texts.page.ts` initial pass поставил импорт `import { httpResource }`
   в НИЗ файла (после class declaration). Это работает в runtime (ESM hoists imports) но
   ESLint `import/first` flag'нул в code-reviewer. Переместил в TOP с другими `@angular/common/http`.

3. **`DialogRef.data` does not exist**: TS2339 «Property 'data' does not exist on type 'DialogRef<X>'.
   Root cause: DialogRef type НЕ имеет поля `.data` (только `.closed` + `.close()`). Dialog data
   хранится в ОТДЕЛЬНОМ токене `PI_DIALOG_DATA = new InjectionToken<unknown>('PI_DIALOG_DATA')`.
   Fixed via `inject(PI_DIALOG_DATA) as TextBlock | null` pattern.

4. **Form value narrowing**: TS2322 + TS18048 на FormArray<TableColumnForm>['value'] — c.key/c.label
   видимы как `string | undefined` потому что FormControl<T> types `.value` как
   `T | null | undefined` per Angular's strict typing. NonNullableFormBuilder гарантирует
   runtime non-null, но TypeScript не видит. Fixed via `?? ''` defaults + `(c.key ?? '')` pattern.
